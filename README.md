# ShopLite — навчальний проєкт

**Курс:** Розробка ПЗ на платформі Node.js  
**Формат:** 2 сервіси × 3 HTTP-роути, CI/CD, тести, діаграми  
**Виконавець:** ІО-36 Білостенний Богдан

## Ідея (ЛР0)

**ShopLite** — спрощений бекенд для інтернет-магазину з двома сервісами:

- **Catalog Service** — керує товарами (каталог).
- **Order Service** — керує замовленнями; для валідації позицій звертається до Catalog Service.

## Репозиторій

- GitHub: `https://github.com/tollkomji/ShopLite`
<!-- - Staging (після ЛР6): `<STAGING_URLS>` -->

## Стек

- Node.js + TypeScript
- Express (HTTP API)
- PostgreSQL + Prisma (після ЛР4)
- Тести: unit / integration / E2E (після ЛР5)
- CI/CD: GitHub Actions + деплой на staging (після ЛР6)

## Структура репозиторію

```
.
├── apps/
│   ├── catalog-service/
│   └── order-service/
├── packages/
│   ├── shared/        # спільні DTO/типи (за потреби)
│   └── config/        # спільні конфіги (eslint/prettier тощо)
└── docs/              # документація + діаграми (ЛР2)
```

## Лабораторна робота №1 — налаштування інструментів

У проєкті налаштовано:

- форматування коду (Prettier)
- лінтинг (ESLint)
- git hooks (Husky + lint-staged)
- стиль комітів (commitlint + conventional commits)

> Детальні команди та вимоги до перевірок будуть закріплені у CI (ЛР6).

## Лабораторна робота №2 — структура застосунку (docs/)

Документація з ЛР2 знаходиться у папці **docs/**:

- `docs/architecture.md` — компоненти, модулі та взаємодія сервісів
- `docs/data-model.md` — модель даних та ER-діаграма
- `docs/scenarios.md` — ключові сценарії, зміни/агрегації даних
- `docs/api.md` — опис HTTP API (ендпоінти + приклади)

## Швидкий старт (після реалізації сервісів)

> Примітка: у межах **ЛР3** було зібрано інтерактивний прототип API на статичних/тестових даних,
> а в **ЛР4** цей прототип замінено на роботу з **PostgreSQL/Prisma**. Поточна версія репозиторію містить реалізацію після ЛР4.

> На етапі ЛР2 тут може бути “каркас”. Після ЛР3/ЛР4 блок буде актуальним.

### Встановлення

```bash
pnpm install
```

### Запуск (приклад)

```bash
# у майбутньому
pnpm -r dev
```

### Перевірки

```bash
pnpm lint
pnpm format:check
pnpm test:unit
pnpm test
pnpm build
```
