# ЛР5 — Звіт з тестування

У проєкті налаштовано 4 рівні перевірок:

- **Unit-тести** — перевіряють доменну логіку без зовнішніх залежностей (наприклад `apps/order-service/src/domain/*`).
- **Інтеграційні тести** — тестують HTTP API з реальним підключенням до PostgreSQL у тестовій схемі (наприклад `catalog_test`).
- **E2E-тести** — піднімають Order Service + stub Catalog Service і перевіряють повний сценарій створення замовлення.
- **Мутаційне тестування (Stryker)** — перевіряє ефективність тестів шляхом внесення штучних змін у код.

## Як запускати

```bash
pnpm install

# Unit (без БД)
pnpm test:unit

# Повний набір (потрібна БД)
# 1) docker compose up -d
# 2) prisma migrate deploy у тестові схеми
pnpm test

# Мутаційні тести
pnpm test:mutation
```

## Висновок

Набір тестів покриває як **чисту бізнес-логіку**, так і **реальну інтеграцію з БД** та **міжсервісну взаємодію** на E2E-рівні.
Мутаційне тестування показує, що більшість "штучних" дефектів ловиться тестами, але при розширенні функціоналу варто
додавати негативні кейси та edge-case перевірки (типи даних, валідації, сценарії помилок БД/мережі).
